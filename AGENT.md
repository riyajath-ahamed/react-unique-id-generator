# AGENT.md - Agent Instructions for `react-unique-id-generator`

## Project Summary

`react-unique-id-generator` is a lightweight, zero-dependency npm package that generates unique, sequential IDs for React applications. It supports React 16.8+ and is used for accessibility (form labels, ARIA attributes) and DOM element identification.

## Setup

```bash
npm install     # Install dependencies
npm run build   # Build CJS + ESM bundles
npm test        # Run all tests
```

## Rules

### Do

- Write all source code in TypeScript with strict mode
- Add tests in `tests/` for any new functionality
- Run `npm test` and ensure all tests pass before submitting changes
- Run `npm run build` to verify the build succeeds
- Call `resetId()` in `beforeEach` blocks to isolate test state
- Use conventional commit messages: `feat:`, `fix:`, `docs:`, `test:`, `chore:`
- Keep changes focused — one feature or fix per PR

### Do Not

- Do not add runtime dependencies — this package must remain zero-dependency
- Do not modify `dist/` — it is generated by the build
- Do not change the public API surface without explicit approval
- Do not use `any` types — prefer proper typing
- Do not skip or disable tests

## Public API

All exports come from `lib/index.ts`:

| Export | Type | Description |
|---|---|---|
| `nextId(localPrefix?)` | Default export | Generate next ID with optional prefix override |
| `generateId(prefix?, suffix?)` | Named export | Generate ID with explicit prefix/suffix (ignores globals) |
| `resetId()` | Named export | Reset counter to 0 |
| `setGlobalPrefix(prefix)` | Named export | Set prefix for all `nextId()` calls |
| `setGlobalSuffix(suffix)` | Named export | Set suffix for all `nextId()` calls |
| `getCurrentId()` | Named export | Get current counter value without incrementing |
| `setId(id)` | Named export | Set counter to specific value (clamps negatives, floors decimals) |

## Design Constraints

- `nextId` is the **default export**; everything else is a **named export**
- The counter (`lastId`) is a **module-scoped singleton** — this is intentional
- `generateId` uses only its own arguments and does **not** read global prefix/suffix
- `nextId` with a local prefix **overrides** the global prefix but still **appends** the global suffix
- `react` is a **peer dependency** (`>=16.8.0`), not a direct dependency

## Testing

- Framework: Jest with ts-jest and jsdom environment
- Test files: `tests/*.test.ts` and `tests/*.test.tsx`
- Coverage: `npm run test:coverage` (reports to `coverage/`)
- CI runs tests on Node.js 18.x and 20.x

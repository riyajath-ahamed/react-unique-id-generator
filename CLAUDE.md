# CLAUDE.md

## Project Overview

`react-unique-id-generator` is a lightweight, zero-dependency npm package for generating unique, sequential IDs in React applications. It targets accessibility use cases (form labeling, ARIA attributes) and works across React 16.8+.

**Published on npm:** `react-unique-id-generator`
**Repository:** `riyajath-ahamed/react-unique-id-generator`

## Commands

```bash
npm test              # Run tests (Jest + jsdom)
npm run test:watch    # Watch mode
npm run test:coverage # Coverage report
npm run build         # Production build (tsup + tsc declarations)
npm run build:watch   # Dev build with watch
npm run clean         # Remove dist/
```

## Architecture

### Source Code

All source lives in `lib/` ‚Äî just two files:

- `lib/nextId.ts` ‚Äî Core module. Contains a module-scoped counter (`lastId`) and all exported functions: `nextId`, `resetId`, `setGlobalPrefix`, `setGlobalSuffix`, `getCurrentId`, `setId`, `generateId`.
- `lib/index.ts` ‚Äî Re-exports everything from `nextId.ts`. This is the build entry point.

### Build Output

Built with **tsup** (esbuild). Config in `tsup.config.ts`. Produces:

- `dist/index.cjs.js` ‚Äî CommonJS bundle (minified)
- `dist/index.esm.js` ‚Äî ES module bundle (minified, tree-shakeable)
- `dist/index.d.ts` ‚Äî Type declarations (generated by `tsc --emitDeclarationOnly`)

`sideEffects: false` is set in `package.json` for tree-shaking.

### Tests

Tests live in `tests/`. Framework: **Jest** with **ts-jest** and **jsdom** environment.

- `tests/nextId.test.ts` ‚Äî Unit tests for all exported functions
- `tests/react-integration.test.tsx` ‚Äî React component integration tests
- `tests/setup.ts` ‚Äî Jest setup (imports `@testing-library/jest-dom`)
- `tests/types.d.ts` ‚Äî Test type declarations

Every test file calls `resetId()` in `beforeEach` to isolate counter state between tests.

### CI/CD

GitHub Actions workflows in `.github/workflows/`:

- `build.yml` ‚Äî Runs on push/PR to `main`, `beta`, `release`. Tests on Node 18.x and 20.x.
- `npm-publish.yml` ‚Äî Publishes to npm on GitHub release creation. Uses `secrets.npm_token`.

## Key Design Decisions

- **Zero runtime dependencies.** Do not add any. `react` is a peer dependency only.
- **Module-scoped mutable state.** The counter (`lastId`), `globalPrefix`, and `globalSuffix` are module-level variables ‚Äî intentionally global singletons.
- **`nextId` is the default export.** All other functions are named exports.
- **Negative values clamped to 0, decimals floored** in `setId()`.
- **`generateId` bypasses global prefix/suffix** ‚Äî it uses only its own arguments.
- **`nextId` with a local prefix overrides global prefix** but still appends global suffix.

## Code Style

- TypeScript strict mode
- No runtime dependencies ‚Äî keep the bundle minimal
- Tests must pass before any PR is merged
- Commit messages follow conventional format: `feat:`, `fix:`, `docs:`, `test:`, `chore:`
